When asked sizing or cost questions, give the bottom-line number first, then brief supporting math. Don't enumerate every field individually — summarize.

**Why:** User asked "how large are our typical transactions?" and expected a quick answer, not a field-by-field breakdown with a full table.

**How to apply:** Lead with the answer ("~1,100 bytes per transaction in Firestore, ~300 bytes as a single encrypted blob"). Only show detailed breakdowns if specifically asked.
